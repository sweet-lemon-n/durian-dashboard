# 看板统计口径

本文档说明「企微智能表」数据源下，看板统计数据从哪里来、怎么算、如何核对。

## 代码位置

- 企微数据聚合主逻辑：`lib/wecom-aggregate.js`
- 手工 JSON 数据源聚合逻辑：`lib/board-routes.js`
- 企微内存缓存：`lib/wecom-cache.js`
- 运行时统计配置：`data/runtime-config.json` 的 `dashboardStats`
- 看板接口：`GET /api/aggregate`
- 后台配置入口：`/admin` → `统计设置`

后台切到「企微智能表」数据源时，`/api/aggregate` 会调用 `lib/wecom-aggregate.js`。

## 使用的企微子表

- `订单主表`
- `分柜明细表`
- `温度记录`
- `陆运明细`

## 订单行分组

一行订单进度按下面维度合并：

- 国家：`订单主表.国家`
- 工厂简称：从 `订单主表.订单编号` 前缀提取，例如 `KK-xxx` 取 `KK`
- 品类：
  - `鲜果(柜数)` 大于 0 生成 `FRESH`
  - `冻果(柜数)` 大于 0 生成 `FROZEN`

同一个国家、工厂简称、品类的多张订单会合并成看板的一行，例如 `KK 鲜果`。

国家维度的订单数按 `订单主表.订单编号` 去重统计。一个订单同时包含鲜果和冻果时，只算 1 张订单，避免国家订单数被品类拆行重复累加。

## 核心指标

### 总订单

同一分组下订单数量。

来源：`订单主表`

### 总箱量

按品类取订单主表里的柜数：

- 鲜果：`订单主表.鲜果(柜数)`
- 冻果：`订单主表.冻果(柜数)`

### 已交付

按 `分柜明细表` 统计：

- 同一个 `订单编号 + 产品品类 + 柜号` 会先去重
- 去重后，`到岸时间` 非空的柜计为已交付

后台 `统计设置` 可以把看板交付率的分子改成：

- 到岸柜数：`到岸时间` 非空
- 签收柜数：`签收日期` 非空
- 发货柜数：`发货日期` 非空
- 分柜登记柜数：分柜明细记录数

### 已签收

按 `分柜明细表` 统计：

- 同一个 `订单编号 + 产品品类 + 柜号` 会先去重
- 去重后，`签收日期` 非空的柜计为已签收

### 在途

按 `分柜明细表` 统计：

- `发货日期` 非空
- 且 `到岸时间` 为空

### 待发

```text
待发 = 订单主表该品类柜数 - 分柜明细表中该订单该品类已登记柜数
```

最小为 0，不会出现负数。

### 交付率

```text
交付率 = 已交付柜数 / 总箱量
```

默认口径：

```text
交付率 = 分柜明细表.到岸时间非空数量 / 订单主表.鲜果(柜数)/冻果(柜数)
```

后台 `统计设置` 可以调整：

- 分子：到岸柜数 / 签收柜数 / 发货柜数 / 分柜登记柜数
- 分母：订单总柜数 / 分柜登记柜数
- 去重：按 `产品品类 + 柜号` 去重，或按记录行统计
- 0 柜数订单行：隐藏或显示

注意：签收数不会再叠加到交付率里，避免超过 100%。

## 物流监控

来源：

- `分柜明细表`
- `温度记录`

### 在途批次

按 `分柜明细表` 每一柜的当前状态统计，当前状态为海外在途的柜计入在途批次。

当前状态由去重后的分柜明细判断：

- 未发货：无 `发货日期`
- 海外在途：有 `发货日期`，无 `到岸时间`
- 已到岸/关口：有 `到岸时间`，无 `陆运日期`，无 `签收日期`
- 国内转运/配送：有 `陆运日期`，无 `签收日期`
- 已签收：有 `签收日期`

温度记录只用于温度展示、平均温度和异常判断，不作为在途批次的主统计来源。

### 温度记录

`温度记录` 总记录数。

### 平均回风温度

按每个柜号最新一条温度记录统计 `回风温度` 平均值。

### 温度异常

按每个柜号最新一条温度记录判断：

```text
abs(回风温度 - 设定温度) > 1.5 => 温度预警
```

如果来源数据已明确标记为 `ALARM`，看板保留为温度异常。

### 关口滞留

按 `分柜明细表` 每一柜的当前状态判断。只有当前状态处于已到岸/关口、尚未进入陆运或签收的柜，才可能计为关口滞留。

默认滞留判断：

- 当前状态为已到岸/关口
- 按到岸后的停留时间计算
- 停留天数达到配置阈值后计入关口滞留

已经进入国内转运或已经签收的柜，不再计入关口滞留。

## 如何人工核对

1. 在企微智能表打开 `订单主表`，按国家、订单编号前缀、品类柜数核对总箱量。
2. 打开 `分柜明细表`，按 `订单编号 + 产品品类 + 柜号` 去重。
3. 对去重后的记录统计：
   - `到岸时间` 非空 = 已交付
   - `签收日期` 非空 = 已签收
   - `发货日期` 非空且 `到岸时间` 为空 = 在途
4. 主看板国家订单数按 `订单主表.订单编号` 去重核对，不按鲜果/冻果拆行重复累加。
5. 物流监控的在途批次和关口滞留按分柜明细当前状态核对；温度记录只核对温度相关指标。
6. 用 `已交付 / 总箱量` 核对交付率。
