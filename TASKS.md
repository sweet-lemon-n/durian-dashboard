# TASKS.md

本文件是项目任务管理中心，用 Markdown Checklist 记录当前任务、待办事项、Bug 和已完成事项。

原则：

- 新任务写到“当前任务”或“待办事项”。
- 已确认的问题写到“Bug 列表”。
- 完成后移动到“已完成事项”并勾选。
- 长期业务知识不要写在这里，应写入 `MEMORY.md`。

## 优先级说明

- `P0`：生产不可用、数据严重错误、安全风险。
- `P1`：核心业务功能异常，影响主要使用流程。
- `P2`：体验问题、性能问题、非阻塞功能缺口。
- `P3`：优化、重构、文档完善。
